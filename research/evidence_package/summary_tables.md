# Reviewer-Facing Summary Tables

This document consolidates the two primary summary tables comparing Sentinel AI's security performance against a standard baseline, and mapping the specific mitigation outcomes for each attack family.

These tables summarize the empirical findings generated by running the reproducibility scripts in [reproducibility.md](../reproducibility.md).

---

## Table 1: Baseline vs. Protected Pipeline Summary
*This table compares the pass-through rate (percentage of payloads that successfully reach the LLM) between an unprotected naive assistant and the 12-layer Sentinel AI pipeline across 35 attack payloads and 4 benign controls.*

| Attack Category | Payload Count | Naive Baseline (Pass-Through) | Protected Pipeline (Pass-Through) | Mitigation Change |
|---|---|---|---|---|
| **Direct Prompt Injection** | 9 | 9 / 9 (100%) | 0 / 9 (0%) | **−100%** |
| **Semantic / Evasion Injection** | 5 | 5 / 5 (100%) | 0 / 5 (0%) | **−100%** |
| **Context Flooding / Token Bombing** | 5 | 5 / 5 (100%) | 0 / 5 (0%) | **−100%** |
| **Token Budget Abuse** | 2 | 2 / 2 (100%) | 0 / 2 (0%) | **−100%** |
| **Behavioral Lockout Probing** | 6 | 6 / 6 (100%) | 0 / 6 (0%) | **−100%** |
| **High-Stakes Actions** | 5 | 5 / 5 (100%) | 0 / 5 (0%) | **−100%** |
| **Privilege Escalation** | 3 | 3 / 3 (100%) | 1 / 3 (33%) ¹ | **−67%** |
| **Benign Control (False Positive Check)** | 4 | 4 / 4 (100%) | 4 / 4 (100%) | **0% (Passed)** |
| **TOTAL (Attacks Only)** | **35** | **35 / 35 (100%)** | **2 / 35 (6%)** | **−94% Pass-through** |

> ¹ **Note on PE-01 (Restricted Document Access):** The request passes through the pipeline, but the confidential document is filtered out of the context by Layer 7 (Context Isolator). The LLM never receives the document content, so the attack fails despite the pipeline technically completing without a block.

## Table 2: Attack Family vs. Mitigating Layer vs. Outcome
*This table tracks exactly which security layer is responsible for intercepting each threat family and the final status of the payload.*

| Attack Family | Case Count | Primary Mitigating Layer | Layer Execution Outcome | Final Payload Status |
|---|---|---|---|---|
| **Direct Injection** | 9 | **Layer 1: Input Validator** | Regular expression block. | **100% Blocked** |
| **Semantic Injection** | 5 | **Layer 2: Semantic Guard** | ML scanner trigger (4 cases) / Regex block (1 case). | **100% Blocked** |
| **Context Flooding** | 5 | **Layer 1 & Layer 4** | Blocked if >10K chars (4 cases) / Truncated (1 case). | **80% Blocked / 20% Truncated** |
| **Budget Abuse** | 2 | **Layer 5: Token Budget** | Daily token budget limit reached. | **100% Blocked** |
| **Behavioral Lockout** | 6 | **Layer 12: Threat Monitor** | Standard regex block (5 cases) $\rightarrow$ Punitive lockout (1 case). | **100% Blocked** |
| **High-Stakes Actions** | 5 | **Layer 11: Human Gate** | Keyword match intercepts action and generates Redis TTL token. | **100% Gated** |
| **Privilege Escalation** | 3 | **Layer 10 & Layer 7** | Identity check block (2 cases) / Document isolation filter (1 case). | **67% Blocked / 33% Filtered** |
| **Benign Control** | 4 | None | No security violations triggered. | **100% Passed** |
